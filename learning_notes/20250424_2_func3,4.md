### Java에서의 람다식
`메서드`를 _하나_ 만 가지고 있는 `인터페이스`에 대해 람다식 적용 가능

```java
@FunctionalInterface
interface 손 {
    잡다(대상);
    // 흔들다(); <- 이거 들어가면 오류, because : @FunctionalInterface
}

손 s = (대상) -> {
        대상을 오른손으로 잡아라;
    }

손 s = new 손(){
        잡다(대상){
            대상을 오른손으로 잡아라
        }
    }
    

static void int main(String[] agrs){
    void test(손){
        s.잡다(펜);
    }
    test((대상) -> {대상을 오른손으로 잡아라;});
}
```

Runnable

```java
ArrayList<Integer> list = List.of(1,2,3,4,5);
list.forEach( (num) -> System.out.println(num)); //에서

// forEach의 동작 과정

public void forEach( Consumer<Integer> c ) {
    for ( T a : this.list )
        c.accept(a);
}

Interface Consumer<T> {
    void accept(T a);
}

// 아래와 같이 익명클래스로 작성해보자면
list.forEach(new Consumer<Integer>() {
    @Override
    public void accept(Integer a) {
        System.out.println(a);
    }
});
```

### python에서의 람다식
```python
# lambda 매개변수: 표현식
def add(num1, num2):
    return num1 + num2
print(add(10, 20))  // 30

add = lambda num1, num2: num1 + num2
print(add(10, 20))  // 30
```

a = int(input("a입력: ")) ## 특정 자료형으로 쓰고 싶으면 강제 형변환 필요

print(number)

number1, number2 = input("두 수 입력: ").split()
print(number1, number2)

# 람다식 변환
number1, number2 = map(lambda n: int(n), input("두 수 입력: ").split())
print(number1, number2)

def parseInt(n):
    return int(n) # 얘도 함수

number1, number2 = map(parseInt, input("두 수 입력: ").split())
print(number1, number2)

# 그러므로
number1, number2 = map(int, input("두 수 입력: ").split()) # int 만 작성해도 함수를 정의한 것과 같으니까 동작이 같음 ㅇㅇ
print(number1, number2)
